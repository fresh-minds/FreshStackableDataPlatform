Hands-on. Doel: één synthetische bron `xyz` aanleveren in de hele keten,
in één klein-maar-volledig PR.

## Spec van de "bron"

We doen alsof beleid een nieuwe stroom `xyz` levert: een gebeurtenis-stream
("event_id, ts, type, value") die elke 5 minuten in batches binnenkomt.
Synthetisch via een generator in `data-generation/`.

## Stap 1 — NiFi-template

Open [NiFi](/go/nifi/), navigeer naar `templates/delta/source-xyz/` (we
kopiëren een bestaand template-paar). Pas processors aan:

- bron-pad in MinIO `s3a://uwv-incoming/xyz/`;
- destination Kafka-topic = `uwv.xyz.event`;
- schema-validatie: `xyz_event.avsc`.

Sla op als nieuwe template-versie.

## Stap 2 — Kafka-topic registreren

In `platform/06-kafka/topics.yaml`:

```yaml
- name: uwv.xyz.event
  partitions: 3
  replicationFactor: 1   # k3d single-node; cloud override apart
  config:
    retention.ms: "604800000"  # 7 dagen
```

`make apply-topics` reconciled.

## Stap 3 — Spark-job

`spark-jobs/streaming-bronze-xyz.yaml` (Stackable SparkApplication-CRD) +
de Python:

```python
# spark-jobs/streaming_bronze_xyz.py
from lib.lakehouse_io import write_delta, kafka_source

stream = (
    kafka_source(topic='uwv.xyz.event', schema_registry='avro:xyz_event')
)
write_delta(
    stream,
    table='bronze.xyz.events',
    path='s3a://uwv-bronze/xyz/',
    checkpoint='s3a://uwv-checkpoints/bronze-xyz/',
    trigger='1 minute',
)
```

`lakehouse_io.write_delta` doet schema-evolutie en checkpoints. Niet
opnieuw uitvinden.

## Stap 4 — Hive-tabel

```sql
CREATE TABLE bronze.xyz.events (
  event_id  STRING,
  ts        TIMESTAMP,
  type      STRING,
  value     DOUBLE
) USING DELTA LOCATION 's3a://uwv-bronze/xyz/';
```

In `platform/05-hive-metastore/init-schemas.sql`.

## Stap 5 — dbt-staging

`dbt/models/staging/stg_xyz.sql`:

```sql
{{ config(materialized='view') }}

SELECT
    event_id,
    ts,
    type,
    value
FROM {{ source('bronze_xyz', 'events') }}
WHERE ts >= dateadd('day', -90, current_date)
```

Plus `schema.yml`:

```yaml
version: 2
sources:
  - name: bronze_xyz
    schema: xyz
    database: bronze
    tables:
      - name: events
models:
  - name: stg_xyz
    meta:
      eigenaar: data.engineer@uwv-platform.local
      classification: intern
    columns:
      - name: event_id
        tests: [not_null, unique]
      - name: ts
        tests: [not_null]
```

## Stap 6 — OPA role-mapping

In `opa-policies-src/data/uwv_role_mappings.json` voeg je `bronze.xyz.*`
toe aan `roles.data_engineer.catalogs_jit` (JIT-only) en `purposes` met
`technisch_onderhoud`.

`make opa-test` om te valideren.

## Stap 7 — OpenMetadata service

`platform/13-openmetadata-config/services/xyz.yaml` voor auto-discovery
van de tabel.

## Stap 8 — Smoke test

`tests/smoke/test_xyz_e2e.py` produceert één event en assert dat het in
`bronze.xyz.events` aankomt binnen 2 minuten.

```bash
make test SUITE=smoke FILTER=xyz
```

## Stap 9 — PR

Eén PR. Reviewers: data-steward (tags + glossary) + platform-admin
(OPA-bundle + smoke-test groen).
