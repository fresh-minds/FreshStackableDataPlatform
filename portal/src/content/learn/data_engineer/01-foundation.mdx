Welkom. Engineer-rol op dit platform combineert vier verantwoordelijkheden:
ingestion (NiFi), streaming compute (Spark), transformatie (dbt), en
orkestratie (Airflow). Plus: discipline rondom bronze-toegang.

## Stackable als basis

Alle data-componenten draaien als CRDs onder de Stackable Data Platform
operators (NiFi, Kafka, Spark, Hive, Trino, Airflow, Superset, OPA, ZooKeeper).
Dat betekent:

- één YAML per cluster (`platform/<nn>-<service>/cluster.yaml`);
- de operator regelt de pods, configmaps, secrets, listeners;
- upgrades zijn declarative — wijzig de release-versie in
  `infrastructure/stackable-release.yaml` en pas toe.

Reden om met Stackable te werken staat in [ADR-0001](/docs/adr/0001-stackable-as-base.md).

## De pipeline

```
Bron (CSV, Kafka, REST, DB)
   │
   ▼
NiFi (ingestion-flow per bron)
   │
   ▼
Kafka topic (uwv.<domein>.<entiteit>)
   │
   ▼
Spark Structured Streaming (write_delta)
   │
   ▼
MinIO (s3a://uwv-bronze/<domein>/)
   │   ◀── Hive Metastore registreert de tabel
   ▼
dbt-trino (staging → silver → gold marts)
   │
   ▼
Trino (queries voor business + Superset)
```

Alles wat je in deze keten doet is versie-beheerd in deze repo.

## Repo-structuur die je moet kennen

| Directory | Wat |
|---|---|
| `platform/` | Stackable CRD's per service, in deploy-volgorde |
| `nifi-flows/templates/` | NiFi-flow-templates per bron |
| `spark-jobs/` | PySpark streaming en batch jobs + helpers |
| `dbt/models/` | staging/intermediate/marts per UC |
| `data-generation/` | Synthetische data-generators |
| `opa-policies-src/` | Rego + role-mappings.json |
| `tests/smoke/` | Smoke-tests die je in CI én lokaal draait |
| `Makefile` | `make cluster`, `make bootstrap`, `make seed`, `make test` |

## JIT-toegang tot bronze

Je werkt 95% van de tijd op silver/gold (dbt-modellen). Voor debug op
bronze (PII zichtbaar) geldt JIT:

```
1. Ticket aanmaken: scope + reden + verwachte duur
2. platform-admin keurt goed → 4 uur toegang
3. queries gelogd; ticket-id verplicht in SQL-comment
4. automatische deactivatie
```

Geen JIT = geen toegang. Geen workarounds — die staan in de audit-log.
