In deze module schrijf je een korte sturingsnotitie op basis van een
afwijking in de lastprognose. Het schrijven is het makkelijke deel; de
data-discipline daarvoor is wat we hier oefenen.

## Casus

Een leidinggevende vraagt: "De voorspelling voor Noord ligt voor december
8% onder de gewone trend. Klopt dat? Wat is er aan de hand?"

## Stap 1 — afwijking valideren

Open [Lastprognose](/go/superset/) → filter regio Noord → december-cohort.
Bevestigt het 8%. Maar: het cijfer kan kloppen *of* het kan een artefact
zijn van data-issues. Volgende stap.

## Stap 2 — DQ-status checken

Open [dbt-docs](/dbt-docs/) → model `gold.uc06_lastprognose` → tab `Tests`.

Drie tests die er nu toe doen:

| Test | Status | Betekenis |
|---|---|---|
| `not_null` op `bedrag_eur` | groen | geen ontbrekende bedragen |
| `unique` op `(maand, regio, uitkeringsoort)` | groen | geen dubbeltellingen |
| `accepted_values` op `uitkeringsoort` | **oranje** | onbekende waarde 'WW-flex-pilot' opgedoken |

De laatste is verdacht. "WW-flex-pilot" is een nieuwe categorie die nog niet
in de glossary staat — die kan voor december juist het verschil maken.

## Stap 3 — lineage volgen

Open [lineage UC-06](/go/openmetadata/). Volg `bedrag_eur` terug naar de bron:

```
bronze.fin.betaling.bedrag → silver.fin.uitkering_betaling.bedrag_eur → gold.uc06_lastprognose.bedrag_eur
```

Op silver staat sinds vorige week een nieuwe `uitkeringsoort` die nog niet
in de mart wordt meegerekend (filter sluit "WW-flex-pilot" uit). Dat
verklaart de 8%.

## Stap 4 — notitie schrijven

```
SUBJECT: Lastprognose Noord december — afwijking verklaard

Cijfer: -8% t.o.v. trend (gold.uc06_lastprognose, snapshot 2025-09-01)

Onderbouwing:
- Niet een werkelijke daling in uitkeringen.
- Sinds week 35 staat in silver.fin.uitkering_betaling een nieuwe
  uitkeringsoort 'WW-flex-pilot' (~12k records). Deze wordt door de
  huidige UC-06 mart nog niet meegerekend (accepted_values-test
  rapporteert oranje sinds 2025-08-30).
- Verwacht herstel zodra data-engineer de mart-definitie aanvult
  (PR uitstaand, ETA: deze week).

Caveats:
- DQ-status onderliggend model: oranje (accepted_values).
- Andere tests groen.

Aanbeveling:
- Geen actie op basis van deze afwijking; melding houden tot mart-fix.
```

Dit is geen bureaucratie. Een lezer ziet meteen: het cijfer klopt
*statistisch*, maar het verhaal erachter is een data-issue, niet een
operationele trend.

## Verificatie

De auto-check controleert dat je notitie minstens vermeldt: tabelnaam,
snapshot-datum, DQ-status, lineage-relatie en aanbeveling.
