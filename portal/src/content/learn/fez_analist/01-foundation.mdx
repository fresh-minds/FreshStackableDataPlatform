Welkom. FEZ werkt met geld op organisatie-niveau, niet op individuele
cliënt-niveau. Je rol is in de breedte: trends, prognoses, afwijkingen.
Het platform is daar precies op ingericht.

## Wat is een gold-mart?

`gold.uc06_lastprognose` is een dbt-model: een afgeleide tabel die elke nacht
opnieuw wordt opgebouwd uit `silver.fin.*`. De mart bevat:

- per maand × regio × uitkeringsoort de uitgekeerde som;
- een 12-maands voortschrijdend gemiddelde;
- de prognose voor de komende 6 maanden (op basis van een eenvoudig ARIMA);
- een DQ-score per record.

Geen PII. Geen BSN's. Geaggregeerd vanaf laag silver.

## Waarom dbt-docs jouw belangrijkste secundaire applicatie is

Voordat je iets uit een dashboard meeneemt naar een rapport: open dbt-docs
en bekijk de testen op het onderliggende model. Drie tests die er bij UC-06
toe doen:

| Test | Wat zegt hij? |
|---|---|
| `not_null` op `bedrag_eur` | Is er een record zonder bedrag? Dan klopt de som niet. |
| `unique` op `(maand, regio, uitkeringsoort)` | Geen dubbeltellingen. |
| `accepted_values` op `uitkeringsoort` | Geen onbekende soorten — anders draait je groupby door. |

De [Data Quality dashboard](/go/openmetadata/) toont dezelfde tests in
groter verband: een 24-uurs trend, falende tests met severity, doorlooptijd
tot fix.

## Lineage in een halve minuut

OpenMetadata laat per kolom zien waar hij vandaan komt. Voor
`gold.uc06_lastprognose.bedrag_eur` zie je:

```
bronze.fin.betaling.bedrag → silver.fin.uitkering_betaling.bedrag_eur → gold.uc06_lastprognose.bedrag_eur
```

Bij elke pijl staat de transformatie (`SUM`, `*1.0`, etc). Voor je in een
overleg een cijfer presenteert, moet je deze keten één keer in je hoofd
hebben — anders is het cijfer een black box.
