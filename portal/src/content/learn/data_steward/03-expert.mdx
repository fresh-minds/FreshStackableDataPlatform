In deze module review je een PR die op het oog klein lijkt maar grote
governance-gevolgen heeft. Doel: leer principle-driven reageren onder
druk.

## De PR

```
PR #5012 — quick: extend crm_medewerker access for handhaving collab
- 3 lines changed in opa-policies-src/data/uwv_role_mappings.json
- author: data.engineer.junior@uwv-platform.local
- urgency: "blocker for project deadline donderdag"

  diff:
  -    "can_see_bankrekening": false,
  +    "can_see_bankrekening": true,    // unmasked IBAN voor crm
```

## Het probleem

Vier dingen die niet kloppen:

1. **Doelbinding overstegen.** crm_medewerker heeft doelbinding `klantcontact`,
   niet `handhaving`. IBAN-unmasking voor klantcontact-doel is geen bestaande
   grondslag — en kan niet zomaar uit een collab-verzoek volgen.
2. **Verkeerde rol gebruikt.** Als handhaving moet kijken: zij hebben
   `ww_handhaver` met al unmasked IBAN. Het probleem is dat hun *werkproces*
   denkt dat ze dat via klantcontact moeten lopen.
3. **Tijdsdruk als argument.** "Project deadline donderdag" is geen
   AVG-uitzondering. Een vraag aan de AVG-coördinator was beter geweest dan
   een quick-PR.
4. **Junior author + geen review-aanvraag.** Een junior heeft niet zomaar
   de scope om dit alleen te doen.

## Hoe je reageert

```markdown
Niet approven. Vragen die ik wil beantwoord hebben voordat dit verder gaat:

1. Wie is de business-eigenaar van dit verzoek? Direct contact met
   crm-team-lead.
2. Wat is het werkproces dat dit veronderstelt? Mogelijk willen we
   het werkproces aanpassen, niet de policies.
3. Heeft de AVG-coördinator hier naar gekeken?
4. Als IBAN echt nodig is in een specifiek scenario: waarom geen
   tijdelijke role-elevation in plaats van permanente uitbreiding?

Geen approval voor donderdag haalbaar; een quick fix op
doelbinding-niveau hoort niet onder tijdsdruk.

Eens met dit standpunt? — let me know.
```

## Wat is de uitkomst

In de praktijk: ofwel het werkproces wordt aangepast (handhaver doet eigen
queries, crm verwijst), ofwel een tijdelijke role-elevation met
verwachtingen-clausule. Niet de doelbinding van crm_medewerker permanent
verbreden.
