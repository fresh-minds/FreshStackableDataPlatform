In deze module loop je één concreet governance-traject door: van PR-melding
tot afgeronde classificatie.

## Casus

In de PR-queue staat:

> #4271 — Add `werkgever_branchecode` to silver.ww.aanvraag

Engineer-comment: "Nieuwe kolom uit polisadm; SBI-code 4-cijferig. Heb
schema.yml aangevuld; geen tag toegevoegd, vraag data-steward."

## Stap 1 — review aanvragen

Open de [tabelpagina](/go/openmetadata/) op `silver.ww.aanvraag`. Klik
"Nieuwe kolom: branchecode" → tab "Tags". Leeg.

Zonder tag heeft OPA geen reden om de kolom te beschermen — hij is dan
ineens zichtbaar voor *alle* WW-rollen. Vaak is dat fine (interne
metadata), maar je moet het bewust doen.

## Stap 2 — classificeren

Branchecode is geen PII (cliënt-attribuut). Het is een attribuut van de
*werkgever*. Tags die passen:

- `bedrijfsdata`
- `intern_gebruik`

Geen `PII`, geen `medisch`, geen `bsn`. Dit zijn de tags die OPA
gebruikt voor automatische masking.

## Stap 3 — glossary

Maak een glossary-term aan — als die nog niet bestaat:

```
Term: branchecode (SBI)
Definitie: Vier-cijferige code volgens de Standaard Bedrijfsindeling van
   het CBS (2008-versie), aangeleverd door werkgever bij polisinschrijving.
Bron: silver.ww.aanvraag.werkgever_branchecode
Eigenaar: data.steward@uwv-platform.local
Use cases: UC-03 (handhaving), UC-06 (lastprognose) — branche-trends
```

## Stap 4 — DQ-tests checken

In dbt-docs: open `stg_ww_aanvraag` → tab Tests. Engineer heeft toegevoegd:

- `not_null` op `werkgever_branchecode` — goed
- `relationships` naar `seed_sbi_codes` — goed (volledigheidscheck)

Voor jou geen verdere actie hier, maar je hebt het wel gezien.

## Stap 5 — OPA role-mappings

Open
[uwv_role_mappings.json](https://vscode.dev/github.com/UWV/UDP_Stackable/blob/main/opa-policies-src/data/uwv_role_mappings.json)
in vscode.dev. Onder `roles.ww_handhaver.unmasked_columns` voeg je:

```json
"silver.ww.aanvraag.werkgever_branchecode"
```

Voor `crm_medewerker` is de kolom niet relevant — niet toevoegen. Default
is "niet zichtbaar" voor onbekende rollen, dat is bewust restrictief.

## Stap 6 — review afsluiten

Comment op de PR:

```
Approved.
Tags: bedrijfsdata, intern_gebruik
Glossary: branchecode (SBI) — koppeling toegevoegd
OPA: ww_handhaver krijgt unmasked toegang; andere rollen niet
DQ: not_null + relationships geconfigureerd door @engineer
```

## Verificatie

De auto-check valideert dat de kolom getagd is, een glossary-term heeft
en in role-mappings.json correct is verwerkt.
