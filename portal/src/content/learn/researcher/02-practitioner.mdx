In deze module doorloop je één compleet, reproduceerbaar mini-onderzoek.
Doel: het volledige patroon onder je vingers — niet de inhoud, maar de
discipline.

## Onderzoeksvraag

> Is er een verschil in re-integratie-uitkomst tussen cliënten met
> uitkeringsoort A en B in de leeftijdsklasse 35–54 (geanonimiseerd)?

## Stap 1 — sandbox query

Open [Trino](/go/trino/) en draai:

```sql
-- ETHICS-2025-103 (onderzoek) — onderzoeksvraag goedgekeurd in commitee
WITH cohort AS (
    SELECT
        pseudo_id,
        regio_klasse,
        leeftijdsklasse,
        uitkeringsoort,
        re_integratie_uitkomst
    FROM sandbox.research.cohort_2025
    WHERE leeftijdsklasse IN ('35-44', '45-54')
      AND uitkeringsoort  IN ('A', 'B')
      AND as_of_date = DATE '2025-09-01'   -- vast datum-filter
)
SELECT
    leeftijdsklasse,
    uitkeringsoort,
    COUNT(*)                                 AS n,
    AVG(CASE WHEN re_integratie_uitkomst='positief' THEN 1.0 ELSE 0.0 END) AS positief_pct
FROM cohort
GROUP BY 1, 2
ORDER BY 1, 2;
```

Twee dingen om op te merken: `pseudo_id` (geen BSN), en
`as_of_date='2025-09-01'` (geen `current_date`).

## Stap 2 — k-check

Voor publicatie controleer je elke groep op n ≥ 10:

| leeftijdsklasse | uitkeringsoort | n | check |
|---|---|---|---|
| 35-44 | A | 1247 | ok |
| 35-44 | B | 38 | ok |
| 45-54 | A | 853 | ok |
| 45-54 | B | 8 | **fail** |

Eén groep heeft n=8. Twee opties:

- **Combineren** (35–54 als één klasse), of
- **Weglaten** (in tekst noemen dat de groep niet rapporteerbaar was).

Voor dit onderzoek combineren we — verder breekt de vergelijking.

## Stap 3 — notebook

Open de [notebook-template in vscode.dev](https://vscode.dev/) en zet
de query erin als reproduceerbaar Python-blok:

```python
import pandas as pd
from trino.dbapi import connect

QUERY = """..."""   # zoals hierboven, met as_of_date hard
df = pd.read_sql(QUERY, connect(host='trino', port=8443))
df_combined = df.groupby('uitkeringsoort').agg(
    n=('n', 'sum'),
    positief_pct=('positief_pct', 'mean'),  # gewogen kan beter — exercise
)
assert (df_combined['n'] >= 10).all(), 'k-anonimiteit gebroken'
print(df_combined)
```

Geen credentials hardcoded — gebruik `KEYCLOAK_*` env vars uit de
devcontainer.

## Stap 4 — mini-rapport

```
SUBJECT: Re-integratie uitkomst per uitkeringsoort, 35-54 jaar
(reproduceerbaar)

Bron:
  sandbox.research.cohort_2025
  dbt-tag: research_2025_q3
  as_of_date: 2025-09-01
  pseudonimisering: pseudo-v3

k-check: alle groepen ≥ 10 (na combineren 35-44 en 45-54).

Resultaat:
  uitkeringsoort A: positief 64% (n=2100)
  uitkeringsoort B: positief 51% (n=46)
Verschil: 13 procentpunt, statistische significantie te toetsen.

Caveats:
- Klein cohort B; herhalen op grotere dataset aanbevolen.
- Combinatie van leeftijdsklasses ivm k-anonimiteit; verlies van
  granulariteit.
```

## Verificatie

De auto-check controleert dat je query een vast `as_of_date` heeft, dat
de k-check aantoonbaar is uitgevoerd, en dat het rapport dbt-tag +
pseudonimisering-versie noemt.
